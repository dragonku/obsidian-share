# Obsidian Canvas 활용 사례

> Canvas = Obsidian의 무한 2D 보드. 노트·이미지·웹페이지·코드를 자유롭게 배치·연결

---

## 1. 프로젝트 전체 구조 시각화

```
[목표] → [마일스톤1] → [마일스톤2] → [완료]
           ↓
        [태스크 노트들]
```

- 프로젝트 목표 카드 → 하위 태스크 노트 연결
- 진행 상태별 컬럼 분류 (Kanban 변형)
- 의존 관계를 화살표로 명시

---

## 2. 독서 맵 / 지식 지도

- 책 챕터별 핵심 인용구 카드 배치
- 개념 간 관계를 선으로 연결
- 여러 책의 공통 개념 수렴 지점 표시
- 예: 「[챕터1 노트] ─ 연관 ─ [챕터3 노트]」

---

## 3. 아이디어 브레인스토밍 보드

- 중앙에 핵심 질문 카드
- 방사형으로 아이디어 카드 펼치기
- 색상으로 카테고리 구분 (기술/비용/리스크)
- 마인드맵보다 자유로운 배치 가능

---

## 4. Daily / Weekly 리뷰 대시보드

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  이번 주 목표 │  │  완료한 것   │  │  다음 주 계획 │
└─────────────┘  └─────────────┘  └─────────────┘
       ↓
┌─────────────────────────────────┐
│         Daily Note 모음          │
└─────────────────────────────────┘
```

- 각 Daily Note를 캔버스에 임베드
- 주간 회고를 한눈에 조망

---

## 5. 시스템 아키텍처 / 플로우차트

- 소프트웨어 컴포넌트 구조도
- API 흐름 다이어그램
- 코드 파일 노트를 실제 연결선으로 이어서 문서화
- 기술 스택 비교표 (카드 나란히 배치)

---

## 6. 연구 / 논문 리뷰 보드

- 논문별 카드: 제목·방법론·결과·한계
- 방법론 유사 논문끼리 그룹핑
- 시간순 배치로 연구 흐름 파악
- 인용 관계를 선으로 표현

---

## 7. 의사결정 프레임워크

```
[선택지 A]          [선택지 B]
  장점 카드            장점 카드
  단점 카드            단점 카드
  리스크 카드          리스크 카드
        ↘          ↙
         [최종 결정]
```

- SWOT / 장단점 비교를 시각적으로
- 결정 근거를 노트로 남겨 추후 회고

---

## 8. 학습 로드맵

- 단계별 학습 노드 배치
- 완료 / 진행 중 / 예정 상태 색상 구분
- 각 노드에 실제 학습 노트 연결
- 선수 지식 의존관계 화살표 표시

---

## 9. 팀 지식 공유 (멀티플레이어 대용)

- 공유 Vault에서 팀원별 작업 영역 분리
- 프로젝트 진행 상황판으로 활용
- 온보딩 가이드 캔버스 (새 팀원용 시각적 안내)

---

## 10. 콘텐츠 파이프라인 관리

```
[아이디어] → [초안] → [검토] → [발행] → [홍보]
```

- 블로그·뉴스레터·유튜브 영상 제작 워크플로우
- 각 단계에 실제 노트 연결
- 플랫폼별 채널 구분

---

## Canvas 핵심 팁

| 팁 | 방법 |
|----|------|
| 노트 임베드 | 기존 `.md` 파일을 캔버스에 드래그 |
| 웹 임베드 | URL 카드로 웹페이지 직접 표시 |
| 그룹 박스 | 카드 여러 개 선택 → 그룹으로 묶기 |
| 색상 구분 | 카드/선에 6가지 색상 지정 가능 |
| 좁히기/넓히기 | 카드 클릭 → 펼쳐서 노트 직접 편집 |
| 링크 방향 | 화살표 방향으로 흐름/인과 표현 |
